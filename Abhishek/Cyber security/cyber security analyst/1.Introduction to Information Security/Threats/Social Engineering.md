## 🎭 What is Social Engineering?

- **Definition** → Psychological manipulation to trick people into revealing information or performing insecure actions.
    
- **Focus** → Exploits the **human element**, often the weakest link in security.
    
- **Analogy** → Con artist at a train station pretending to be trustworthy to steal your belongings.
    

## 🧩 Common Techniques

1. **Phishing** → Fake emails/messages (e.g., bank email asking for login).
    
2. **Pretexting** → Fabricated scenario (e.g., fake IT support call asking for credentials).
    
3. **Baiting** → Enticing trap (e.g., infected USB labeled “Employee Salaries”).
    
4. **Tailgating** → Attacker follows authorized person into secure area.
    
5. **Quid Pro Quo** → Offering benefit in exchange for info (e.g., free software upgrade for login details).
    

## 📉 Impact

- **Data Breaches** → Unauthorized access to sensitive info.
    
- **Financial Losses** → Fraud, theft, monetary damages.
    
- **Reputational Damage** → Loss of customer trust, brand harm.
    
- **Operational Disruption** → Downtime, productivity loss.
    
- **Danger** → Bypasses technical defenses by exploiting human trust.

```
# 1. Phishing
def phishing():

    capture_credentials()

# 2. Pretexting
def pretexting():
    pretend_role("IT_support")
    request_login_details()

# 3. Baiting
def baiting():
    drop_usb("Employee_Salaries_2023")
    install_malware_on_use()

# 4. Tailgating
def tailgating():
    follow_employee_into_secure_area()

# 5. Quid Pro Quo
def quid_pro_quo():
    offer_fake_upgrade()
    request_credentials()

# Overall Social Engineering Attack
def social_engineering():
    phishing()
    pretexting()
    baiting()
    tailgating()
    quid_pro_quo()

social_engineering()
```

## 🎯 Key Takeaway

- Social engineering = **human hacking**.
    
- Even strong technical defenses can fail if **people are tricked**.
    
- Defense requires **awareness training, strict policies, and vigilance**.
