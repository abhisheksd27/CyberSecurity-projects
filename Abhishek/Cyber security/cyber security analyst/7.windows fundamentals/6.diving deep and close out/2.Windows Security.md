## 🆔 Security Identifier (SID)

Every user, group, or process in Windows is uniquely identified using a **SID (Security Identifier)**.

👉 Think of SID like:

> “Actual identity behind the username”

Even if usernames are same → SIDs are always unique.

### Example:

S-1-5-21-674899381-4069889467-2080702030-1002

### Breakdown:

|Part|Meaning|
|---|---|
|S|Identifier (it's a SID)|
|1|Revision|
|5|Authority (Windows)|
|21|Domain/local system|
|long numbers|Machine/domain ID|
|1002|RID (user ID)|

👉 **Important:**

- RID `500` → Administrator
- RID `501` → Guest

## 🔑 Access Control Model (ACL, ACE, Tokens)

Windows security is based on:

### 1. ACL (Access Control List)

- List of permissions attached to an object (file, service, registry, etc.)

### 2. ACE (Access Control Entry)

- Each entry inside ACL
- Defines:
    - Who (user/group)
    - What (permissions)

### 3. Access Token

When a user logs in:

- Windows creates a **token**
- Contains:
    - SID
    - Group SIDs
    - Privileges

👉 Every action = checked against this token

## 🧠 SAM (Security Accounts Manager)

- Local database storing:
    - User accounts
    - Password hashes

📍 Location:

C:\Windows\System32\Config\SAM

👉 Used when:

- System is NOT in domain
- Local authentication

---

## ⚠️ UAC (User Account Control)

UAC prevents unauthorized system changes.

### How it works:

- Even admin runs as **standard user**
- When elevated action needed → prompt appears

👉 Example:

- Installing software
- Modifying system files

### Modes:

- Admin Approval Mode
- Credential Prompt (for standard users)

👉 **Security Insight:**  
Attackers try:

- UAC bypass techniques
- Social engineering (click “Yes”)

## 🗂️ Windows Registry

The **Registry** is like:

> 🧠 “Brain of Windows”

Stores:

- OS settings
- Application configs
- Security configs

---

### 🔑 Important Root Keys

|Key|Purpose|
|---|---|
|HKLM|System-wide settings|
|HKCU|Current user settings|
|HKCR|File associations|
|HKU|All user profiles|

---

### 📁 Important Files

C:\Windows\System32\Config\

Contains:

- SAM
- SYSTEM
- SECURITY
- SOFTWARE

---

## 🚨 Persistence via Registry (VERY IMPORTANT)

Attackers often use:

### Run Keys:

HKLM\Software\Microsoft\Windows\CurrentVersion\Run  
HKCU\Software\Microsoft\Windows\CurrentVersion\Run

👉 Programs here run at startup

### RunOnce Keys:

- Run only once

---

### Example:

malware.exe → auto runs on login

👉 Used for:

- Persistence
- Backdoors

## ✅ Application Whitelisting

### Whitelisting:

- Only allowed apps can run

### Blacklisting:

- Block known bad apps

👉 **Whitelisting = more secure**

---

## 🛡️ AppLocker

Microsoft’s whitelisting tool.

Controls:

- Executables
- Scripts
- DLLs
- Installers

### Rules based on:

- File path
- File hash
- Publisher signature

👉 Used in enterprises

---

## ⚡ Real-World Attack Surface Summary

Windows can be attacked via:

### 1. Misconfigurations

- Weak permissions
- Open shares
- Bad services

### 2. Credential Attacks

- LSASS dumping
- SAM extraction

### 3. Persistence

- Registry Run keys
- Scheduled tasks
- Services

### 4. Privilege Escalation

- Service misconfigs
- DLL hijacking
