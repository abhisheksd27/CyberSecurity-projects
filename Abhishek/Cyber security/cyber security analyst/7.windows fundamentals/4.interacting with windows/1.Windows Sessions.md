# 🔑 Types of Windows Sessions

## 1️⃣ Interactive Session

## 📌 What is it?

- A session where **user interacts directly**

---

## ✅ Ways to create:

- Login on computer
- Remote Desktop (RDP)
- Using `runas` command

---

## 💻 Example

runas /user:admin cmd

👉 Opens:

- New session as **admin user**

---

## 🧠 Key Points

- Requires:
    - Username
    - Password 🔐
- User can:
    - Run commands
    - Open apps
    - Control system

---

## 📌 Example Scenario

- You login to laptop → interactive session
- You connect via RDP → interactive session

---

# 👤 Types of Non-Interactive Accounts

---

## 1️⃣ Local System Account

## 📌 Name:

NT AUTHORITY\SYSTEM

## 🔥 Features:

- Most powerful account
- More powerful than admin

👉 Used for:

- Core OS functions
- Critical services

---

## ⚠️ Pentester Note

- If attacker gets SYSTEM access → 💀 **full control**

---

## 2️⃣ Local Service Account

## 📌 Name:

NT AUTHORITY\LocalService

## 📊 Features:

- Low privileges
- Similar to normal user

👉 Used for:

- Limited services

---

## 3️⃣ Network Service Account

## 📌 Name:

NT AUTHORITY\NetworkService

## 📊 Features:

- Low local privileges
- Can access network resources

👉 Used for:

- Network-based services

---

# ⚖️ Comparison Table

|Account|Privilege Level|Usage|
|---|---|---|
|LocalSystem|🔥 Very High|OS & critical services|
|LocalService|Low|Limited services|
|NetworkService|Low + Network|Network services|

---

# 🔥 Pentester Important Points

## 🎯 Why Sessions Matter

- Different sessions = different privileges

---

## 🔍 Attack Ideas

- Steal tokens from sessions
- Hijack active sessions
- Escalate:
    - User → SYSTEM

---

## ⚠️ Key Targets

- SYSTEM session
- Admin session
- RDP sessions

---

# 🧾 Final Summary (Quick Revision)

- Session = user-system connection
- Types:
    - Interactive → user login
    - Non-interactive → background
- Important accounts:
    - SYSTEM → highest
    - LocalService → limited
    - NetworkService → network
