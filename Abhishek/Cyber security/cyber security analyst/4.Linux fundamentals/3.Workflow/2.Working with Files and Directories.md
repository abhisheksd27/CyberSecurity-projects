### 📄 Creating Files & Directories

| Command                    | Description                                                                               |
| -------------------------- | ----------------------------------------------------------------------------------------- |
| `touch <name>`             | Creates a **new empty file** with the given name                                          |
| `touch ./path/to/file.txt` | Creates a file **inside a specific directory** using a relative path                      |
| `mkdir <name>`             | Creates a **new directory**                                                               |
| `mkdir -p path/to/dir`     | Creates a directory and all **parent directories** automatically (no error if they exist) |

---

### 🌳 Viewing Directory Structure

|Command|Description|
|---|---|
|`tree .`|Displays a **visual tree structure** of all files and directories starting from the current directory (`.`)|

---

### 🔀 Moving & Renaming

|Command|Description|
|---|---|
|`mv <source> <destination>`|**Moves** a file or directory to a new location|
|`mv oldname.txt newname.txt`|**Renames** a file by moving it to a new name|
|`mv file1.txt file2.txt dir/`|Moves **multiple files** into a directory at once|

---

### 📋 Copying

|Command|Description|
|---|---|
|`cp <source> <destination>`|**Copies** a file to a specified location (original remains intact)|

---

### 🗑️ Deleting (Exercise — Research These)

| Command              | Description                                                         |
| -------------------- | ------------------------------------------------------------------- |
| `rm <file>`          | **Removes/deletes** a file                                          |
| `rm -r <directory>`  | **Recursively deletes** a directory and all its contents            |
| `rm -rf <directory>` | Force-deletes a directory **without prompting** (use with caution!) |
| `rmdir <directory>`  | Removes an **empty directory** only                                 |
	
---

### 💡 Key Concepts

|Concept|Description|
|---|---|
|`.` (single dot)|Refers to the **current directory** — use it to specify relative paths|
|`mkdir -p`|The `-p` flag stands for **"parents"** — creates the full path in one command|
|`mv` for renaming|There is **no separate rename command** in Linux — `mv` handles both moving and renaming|
|`cp` vs `mv`|`cp` **keeps the original**, `mv` **removes it** from the source location|
|Regex & redirection|Linux allows powerful **batch editing** using regex and output redirection (covered in later sections)|

---

### 🔁 Practical Example Workflow

bash

```bash
touch info.txt                          # Create a file
mkdir Storage                           # Create a directory
mkdir -p Storage/local/user/documents  # Create nested directories
touch ./Storage/local/user/userinfo.txt # Create file inside nested dir
mv info.txt information.txt             # Rename a file
mv information.txt readme.txt Storage/  # Move multiple files
cp Storage/readme.txt Storage/local/    # Copy a file to another location
tree .                                  # View the full structure
```
