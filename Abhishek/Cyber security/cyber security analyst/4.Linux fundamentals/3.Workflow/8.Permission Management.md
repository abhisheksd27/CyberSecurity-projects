## 🌟 Concept

Permissions in Linux are like **keys** that control who can access, modify, or execute files and directories. Each file has:

- **Owner** (user who created it)
    
- **Group** (team associated with it)
    
- **Others** (everyone else)
    

## 📌 Types of Permissions

- **r (Read)** → View contents of a file or list directory contents.
    
- **w (Write)** → Modify file contents or add/delete files in a directory.
    
- **x (Execute)** → Run a file (if it’s a program/script) or traverse a directory.
    

## 🗂 Permission Structure

Example:

bash

```
-rwxrw-r--
```

Breakdown:

- **File type**: `-` (file), `d` (directory), `l` (link)
    
- **Owner permissions**: `rwx` → read, write, execute
    
- **Group permissions**: `rw-` → read, write
    
- **Others permissions**: `r--` → read only
    

## 🔄 Octal Representation

Permissions can also be set using numbers:

|Value|Binary|Meaning|
|---|---|---|
|7|111|rwx|
|6|110|rw-|
|5|101|r-x|
|4|100|r--|

Example:

bash

```
chmod 754 file
```

→ Owner: rwx, Group: r-x, Others: r--

## ⚙️ Changing Permissions

- **Symbolic method**:
    
    bash
    
    ```
    chmod u+x file   # add execute for owner
    chmod g-w file   # remove write for group
    chmod a+r file   # add read for all
    ```
    
- **Octal method**:
    
    bash
    
    ```
    chmod 644 file   # rw- r-- r--
    ```
    

## 👥 Changing Ownership

bash

```
chown user:group file
```

Example:

bash

```
chown root:root shell
```

## 🔑 Special Permissions

### 1. **SUID (Set User ID)**

- Runs program with **owner’s privileges**.
    
- Shown as `s` in place of `x` for owner.
    
- Example risk: If misused, can allow normal users to run commands as root.
    

### 2. **SGID (Set Group ID)**

- Runs program with **group’s privileges**.
    
- Shown as `s` in place of `x` for group.
    

### 3. **Sticky Bit**

- Applied to directories.
    
- Ensures only **file owner, directory owner, or root** can delete/rename files.
    
- Shown as `t` (or `T` if execute not set).
    
- Commonly used in `/tmp`.
    

## 📊 Diagram: Permission Layers

Code

```
File/Directory
   │
   ├── Owner (u)
   ├── Group (g)
   └── Others (o)
        │
        ├── Read (r)
        ├── Write (w)
        └── Execute (x)
```

## 📝 Key Takeaways

- **Execute on directory** = ability to traverse, not to see contents.
    
- **Write on directory** = ability to add/delete files.
    
- **SUID/SGID** = temporary privilege escalation.
    
- **Sticky bit** = protects files in shared directories.
