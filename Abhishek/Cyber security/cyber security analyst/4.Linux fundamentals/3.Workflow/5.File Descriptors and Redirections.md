### 📋 File Descriptors (FD)

|FD|Name|Description|
|---|---|---|
|`0`|**STDIN**|Standard **Input** — data fed into a program|
|`1`|**STDOUT**|Standard **Output** — normal result/output of a program|
|`2`|**STDERR**|Standard **Error** — error messages from a program|

> Think of a file descriptor like a **ticket number** — it's how the OS keeps track of which I/O resource to interact with.

---

### ➡️ Redirection Operators

| Operator      | Description                                                            |
| ------------- | ---------------------------------------------------------------------- |
| `>`           | Redirect **STDOUT** to a file — **overwrites** if file exists          |
| `>>`          | Redirect **STDOUT** to a file — **appends** without overwriting        |
| `2>`          | Redirect **STDERR** to a file                                          |
| `1>`          | Redirect **STDOUT** to a file (explicit)                               |
| `<`           | Redirect a file as **STDIN** input to a command                        |
| `<<`          | **Here-document** — feed a stream of input until EOF marker is reached |
| `2>/dev/null` | Discard all **error messages** (send to null device)                   |

---

### 📌 Redirection Examples

|Command|What it Does|
|---|---|
|`find /etc/ -name shadow 2>/dev/null`|Suppress all **permission denied** errors|
|`find /etc/ -name shadow 2>/dev/null > results.txt`|Save **STDOUT** to file, discard errors|
|`find /etc/ -name shadow 2> stderr.txt 1> stdout.txt`|Save **STDERR and STDOUT** to separate files|
|`cat < stdout.txt`|Use file contents as **STDIN**|
|`find /etc/ -name passwd >> stdout.txt 2>/dev/null`|**Append** results to existing file|
|`cat << EOF > stream.txt`|Stream input into a file until **EOF** is typed|

---

### 🔗 Pipes

|Operator|Description|
|---|---|
|`\|`|**Pipe** — takes STDOUT of one command and feeds it as STDIN to the next|

#### Pipe Examples

|Command|What it Does|
|---|---|
|`find /etc/ -name *.conf 2>/dev/null \| grep systemd`|Find `.conf` files then **filter** results containing "systemd"|
|`find /etc/ -name *.conf 2>/dev/null \| grep systemd \| wc -l`|Find → filter → **count** the number of results|

---

### 🛠️ Key Tools Used with Redirection

|Tool|Description|
|---|---|
|`grep <pattern>`|**Filters** output — only shows lines matching the pattern|
|`wc -l`|**Counts** the number of lines in the output|
|`/dev/null`|The **null device** — discards anything sent to it|
|`cat`|Reads file contents or standard input and prints to STDOUT|
|`EOF`|**End-Of-File** marker used with `<<` to signal end of input stream|

---

### 💡 Key Concepts

|Concept|Description|
|---|---|
|`>` vs `>>`|`>` **overwrites**, `>>` **appends** — always use `>>` to preserve existing data|
|Chaining pipes|You can chain **multiple pipes** together: `cmd1 \| cmd2 \| cmd3`|
|FD numbers|You can explicitly use `1>` and `2>` to control **exactly which stream** gets redirected|
|`2>/dev/null`|One of the most commonly used tricks to **clean up output** by hiding errors|
