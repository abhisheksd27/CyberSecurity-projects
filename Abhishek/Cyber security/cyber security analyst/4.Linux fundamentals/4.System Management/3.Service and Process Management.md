## 🧠 Summary Notes: Service & Process Management in Linux

### 🔧 Services (Daemons)

- Run in background, no direct user interaction.
    
- Identified by `d` suffix (e.g., `sshd`, `systemd`).
    

#### Types:

|Type|Description|Example|
|---|---|---|
|System Services|Start at boot, essential for OS|`systemd`, `networkd`|
|User-Installed|Added by user, enhances functionality|`apache2`, `sshd`|

#### Goals:

- Start/Restart
    
- Stop
    
- Check status/logs
    
- Enable/Disable on boot
    
- Find service/process
    

#### 🔍 Commands:

bash

```
systemctl start ssh
systemctl status ssh
systemctl enable ssh
systemctl list-units --type=service
journalctl -u ssh.service --no-pager
```

### ⚙️ Process Management

#### 🧬 Process States:

- Running
    
- Waiting
    
- Stopped
    
- Zombie
    

#### 🔫 Kill Commands:

| Command   | Use Case                 |
| --------- | ------------------------ |
| `kill`    | Send signal to PID       |
| `pkill`   | Kill by name             |
| `pgrep`   | Find process by name     |
| `killall` | Kill all with given name |

#### 🔔 Common Signals:

|Signal|Description|
|---|---|
|SIGHUP|Terminal closed|
|SIGINT|Ctrl + C|
|SIGQUIT|Ctrl + D|
|SIGKILL|Force kill|
|SIGTERM|Graceful termination|
|SIGSTOP|Stop (non-recoverable)|
|SIGTSTP|Suspend (Ctrl + Z)|

**Example:**

bash

```
kill -9 <PID>  # Force kill
```

### 🪄 Background & Foreground

#### Suspend & Resume:

bash

```
ping -c 10 www.hackthebox.eu
[Ctrl + Z]       # Suspend
jobs             # List background jobs
bg               # Resume in background
fg 1             # Bring job 1 to foreground
```

#### Run in Background:

bash

```
ping -c 10 www.hackthebox.eu &
```

### 🔗 Multiple Commands

| Separator | Behavior                           | Example                           |           |              |
| --------- | ---------------------------------- | --------------------------------- | --------- | ------------ |
| `;`       | Run all, ignore errors             | `echo '1'; ls missing; echo '3'`  |           |              |
| `&&`      | Run next only if previous succeeds | `echo '1' && ls file && echo '3'` |           |              |
| `         | `                                  | Pipe output to next command       | `cat file | grep 'text'` |
