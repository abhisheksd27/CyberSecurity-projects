## 🛠️ First Steps with Commands

- **Example**:
    
    Code
    
    ```
    ls
    ```
    
    → Lists files and directories in the current folder.
    
- **Options**: Commands often have optional parameters (flags) to change behavior. Example:
    
    Code
    
    ```
    ls -la
    ```
    
    → Shows all files (including hidden ones) in long format.

## 📖 How to Get Help

### 1. **Man Pages (Manuals)**

- Command:
    
    Code
    
    ```
    man <tool>
    ```
    
- Example:
    
    Code
    
    ```
    man ls
    ```
    
- Provides **detailed documentation**: syntax, options, description.
    
- Navigation: `Space` (scroll), `q` (quit), `h` (help).
    

### 2. **--help Option**

- Command:
    
    Code
    
    ```
    <tool> --help
    ```
    
- Example:
    
    Code
    
    ```
    ls --help
    ```
    
- Shows **quick usage summary** + available options.
    
- Faster than reading full man pages.
    

### 3. **-h Option**

- Some tools (like `curl`) use `-h` for short help.
    
- Example:
    
    Code
    
    ```
    curl -h
    ```
    

### 4. **apropos**

- Searches man page descriptions for keywords.
    
- Command:
    
    Code
    
    ```
    apropos <keyword>
    ```
    
- Example:
    
    Code
    
    ```
    apropos sudo
    ```
    
- Useful when you don’t know the exact command name.
    

### 5. **explainshell.com**

- Web tool: paste a command, get explanations for each part.
    
- Great for **long or complex commands**.
    

## 🔄 Flow Diagram – Getting Help

Code

```
Need info → man <tool> (full manual)
         → <tool> --help (quick summary)
         → <tool> -h (short help, some tools)
         → apropos <keyword> (search commands)
         → explainshell.com (web breakdown)
```

## 🌍 Real-World Example

- You see `ls -la /etc` in a script.
    
- Unsure what `-la` means?
    
    - Run `man ls` → detailed explanation.
        
    - Or `ls --help` → quick option list.
        
- If you don’t know `ls` exists, run `apropos list` → shows related commands.
    

## 🔑 Key Takeaway

Learning commands is **exploration-driven**. You don’t need to memorize everything — instead, know how to **find help quickly**:

- `man` for deep dive.
    
- `--help` for quick glance.
    
- `apropos` for discovery.
    
- `explainshell.com` for breakdowns.
