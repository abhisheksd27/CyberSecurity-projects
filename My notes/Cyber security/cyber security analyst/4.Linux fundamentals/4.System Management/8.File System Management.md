## 📘 File System Types

- **ext2** → no journaling, low overhead (USB drives).
    
- **ext3/ext4** → journaling, reliable, ext4 = default modern choice.
    
- **Btrfs** → snapshots, integrity checks, advanced storage setups.
    
- **XFS** → high performance, large file handling.
    
- **NTFS** → Windows compatibility (dual-boot, external drives).
    

## 🧠 Inodes

- Store **metadata**: permissions, ownership, size, timestamps.
    
- Do **not** store file name/data.
    
- Inode table = catalog of all files.
    
- **Analogy**: Inodes = library index cards; inode table = catalog.
    

### Example

bash

```
ls -il
10678872 -rw-r--r-- 1 user group 234123 Feb 14 19:30 myscript.py
```

- Shows inode number + metadata.
    

## 📁 File Types

- **Regular files** → text, binary, executables.
    
- **Directories** → containers for files/subdirectories.
    
- **Symbolic links (symlinks)** → shortcuts to files/dirs.
    

## 🔑 Permissions

- Owner, group, others → independent permission sets.
    
- Example: `-rw-r--r--` → owner read/write, group read, others read.
    

## 💽 Disk Management

- Tools: `fdisk`, `gpart`, `GParted`.
    
- Partitioning → divide physical storage into logical sections.
    
- Each partition formatted with FS (ext4, NTFS, FAT32).
    

### Example: View Partitions

bash

```
sudo fdisk -l
```

Shows disk size, partitions, types.

## 📌 Mounting

- Mount = link partition/drive to directory (mount point).
    
- Auto-mount at boot → `/etc/fstab`.
    

### Example: Mount USB

bash

```
sudo mount /dev/sdb1 /mnt/usb
cd /mnt/usb && ls -l
```

### Example: Unmount

bash

```
sudo umount /mnt/usb
```

- Cannot unmount if in use → check with:
    

bash

```
lsof | grep /mnt/usb
```

### Example: fstab entry

txt

```
/dev/sdb1 /mnt/usb ext4 rw,noauto,user 0 0
```

## 🌀 Swap Space

- Extends RAM when full → inactive pages moved to swap.
    
- Tools: `mkswap`, `swapon`.
    
- Size depends on RAM & workload.
    
- Recommended to encrypt swap (sensitive data).
    
- **Hibernation** → saves system state to swap, restores on boot.
    

## 🧠 Key Takeaways

- Choose FS based on **performance, reliability, compatibility**.
    
- Inodes = metadata backbone of Linux FS.
    
- Mounting links storage → accessible in hierarchy.
    
- Swap ensures smooth performance & supports hibernation.
