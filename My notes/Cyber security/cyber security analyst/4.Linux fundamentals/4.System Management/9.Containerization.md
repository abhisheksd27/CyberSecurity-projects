## 📘 Concept

- **Containerization** = packaging apps into isolated environments called **containers**.
    
- Containers share the host’s kernel → lighter than virtual machines.
    
- Benefits: **portable, secure, scalable, consistent** across dev/test/prod.
    

**Analogy:** Like giving each band at a concert its own portable stage pod. They share the main stage but don’t interfere with each other.

## 🔒 Security

- Containers isolate apps from host & each other.
    
- Safer than running directly on host, but **less isolation than VMs**.
    
- Risks: privilege escalation, container escape if misconfigured.
    

## 🐳 Docker

- Most popular container platform.
    
- Uses **images** (templates) → run as **containers** (instances).
    
- **Docker Hub** = library of images (public & private).
    

**Analogy:** Docker = lunchbox. Each container = sealed lunchbox with food (app). To change contents, make a new recipe (Dockerfile).

### Install Docker

bash

```
sudo apt install docker-ce docker-ce-cli containerd.io -y
docker run hello-world
```

### Dockerfile Example

dockerfile

```
FROM ubuntu:22.04
RUN apt-get update && apt-get install -y apache2 openssh-server
RUN useradd -m docker-user && echo "docker-user:password" | chpasswd
EXPOSE 22 80
CMD service ssh start && /usr/sbin/apache2ctl -D FOREGROUND
```

### Build & Run

bash

```
docker build -t FS_docker .
docker run -p 8022:22 -p 8080:80 -d FS_docker
```

- Maps host ports → container ports.
    
- Access SSH on port 8022, Apache on port 8080.
    

### Management Commands

- `docker ps` → list running containers.
    
- `docker stop <id>` → stop container.
    
- `docker start <id>` → start container.
    
- `docker logs <id>` → view logs.
    
- `docker rm <id>` → remove container.
    

⚠️ Containers are **stateless** → changes lost when stopped. Use **volumes** for persistent data.

## 🐧 LXC (Linux Containers)

- System-level containerization.
    
- Uses **cgroups** + **namespaces** for isolation.
    
- More like lightweight VMs.
    

### Install & Create

bash

```
sudo apt install lxc -y
sudo lxc-create -n linuxcontainer -t ubuntu
```

### Manage

- `lxc-ls` → list containers.
    
- `lxc-start -n name` → start.
    
- `lxc-stop -n name` → stop.
    
- `lxc-attach -n name` → connect to container.
    

### Security

- Restrict access (disable SSH, limit IPs).
    
- Limit resources with cgroups:
    

txt

```
lxc.cgroup.cpu.shares = 512
lxc.cgroup.memory.limit_in_bytes = 512M
```

- Restart service:
    

bash

```
sudo systemctl restart lxc.service
```

## 🔍 Docker vs LXC

|Feature|Docker (App-focused)|LXC (System-focused)|
|---|---|---|
|Approach|Package apps/microservices|Isolated Linux environments|
|Images|Standardized Docker images|Manual root filesystem setup|
|Portability|High (Docker Hub)|Lower, tied to host|
|Ease of use|Simple CLI, beginner-friendly|Needs Linux admin knowledge|
|Security|Extra isolation (AppArmor, SELinux)|Secure but needs config|

## 🧠 Key Takeaways

- Containers = lightweight, portable, isolated app environments.
    
- Docker → best for apps & microservices.
    
- LXC → best for system-level isolation.
    
- Always secure containers: restrict access, limit resources, keep updated.
