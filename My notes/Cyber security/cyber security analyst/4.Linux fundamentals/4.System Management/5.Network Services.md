# 🌐 Network Services in Linux

## 📘 Concept

- Network services = background programs enabling communication, file transfer, remote access.
    
- Important for **admins** (secure configuration) & **penetration testers** (detect vulnerabilities).
    
- Example risk: **FTP** transmits credentials in plain text → attacker can capture them.
    
- Secure alternatives (SSH, NFS with proper configs) reduce exposure.
    

## 🔑 SSH (Secure Shell)

- **Purpose**: Secure remote login, command execution, file transfer.
    
- **Server**: OpenSSH (most common implementation).
    
- **Encryption**: Protects data from interception.
    

### Install & Check Status

bash

```
sudo apt install openssh-server -y
systemctl status ssh
```

**Example Output:**

- Service: `ssh.service`
    
- Active: running
    
- Main PID: `sshd`
    

### Connect to Remote Host

bash

```
ssh user@10.129.17.122
```

- Prompts for password.
    
- Adds host fingerprint to known hosts.
    

### Configuration

- File: `/etc/ssh/sshd_config`
    
- Options: password vs key login, max connections, host key checking.
    
- ⚠️ Must edit carefully to avoid locking out access.
    

**Use Cases:**

- Secure login for admins.
    
- Port forwarding/tunneling for encrypted data transfer.
    

## 📂 NFS (Network File System)

- **Purpose**: Share/manage files across systems as if local.
    
- **Servers**: `nfs-utils` (Ubuntu), `NFS-Ganesha` (Solaris), `OpenNFS` (RedHat).
    
- **Features**: central storage, collaboration, replication, access controls.
    

### Install & Check Status

bash

```
sudo apt install nfs-kernel-server -y
systemctl status nfs-kernel-server
```

**Example Output:**

- Service: `nfs-server.service`
    
- Active: running/exited successfully.
    

### Configuration File

- Path: `/etc/exports`
    
- Defines shared directories + permissions.
    

#### Common Permissions:

|Option|Description|
|---|---|
|rw|Read + Write access|
|ro|Read-only access|
|no_root_squash|Root on client = root on server|
|root_squash|Restrict client root to normal user|
|sync|Data written only after saved|
|async|Faster, but risk of inconsistency|

### Example: Create NFS Share

bash

```
mkdir ~/nfs_sharing
echo '/home/user/nfs_sharing hostname(rw,sync,no_root_squash)' >> /etc/exports
cat /etc/exports
```

**Output:**

Code

```
/home/user/nfs_sharing hostname(rw,sync,no_root_squash)
```

### Mount NFS Share (on client)

bash

```
mount hostname:/home/user/nfs_sharing /mnt
```


# 🌐 Network Services in Linux (Part 2)

## 📂 Mounting NFS Share

- **Purpose**: Access remote files locally as if they were on your system.
    
- **Example**:
    

bash

```
mkdir ~/target_nfs
mount 10.129.12.17:/home/john/dev_scripts ~/target_nfs
tree ~/target_nfs
```

**Output:**

Code

```
target_nfs/
├── css.css
├── html.html
├── javascript.js
├── php.php
└── xml.xml
```

- Files from remote NFS share now accessible locally.
    
- ⚠️ Misconfigured NFS can allow **privilege escalation**.
    

## 🌍 Web Servers

### Concept

- Deliver data/apps via **HTTP/HTTPS**.
    
- Common Linux servers: **Apache, Nginx, Lighttpd, Caddy**.
    
- Used for hosting websites, file transfers, phishing simulations, vulnerability testing.
    

### Apache Web Server

- **Install**:
    

bash

```
sudo apt install apache2 -y
```

- **Config file**: `/etc/apache2/apache2.conf`
    
- Example config:
    

apache

```
<Directory /var/www/html>
    Options Indexes FollowSymLinks
    AllowOverride All
    Require all granted
</Directory>
```

- Place files in `/var/www/html` → download via `wget` or `curl`.
    
- **.htaccess** → directory-level settings (access control, overrides).
    
- **Modules**: `mod_rewrite`, `mod_security`, `mod_ssl` for added security.
    

### Python Web Server

- Lightweight alternative for quick file hosting.
    
- **Install & Run**:
    

bash

```
sudo apt install python3 -y
python3 -m http.server        # Default port 8000
python3 -m http.server 443    # Custom port
python3 -m http.server --directory /home/user/files
```

- Access via browser → download files directly.
    

## 🔒 VPN (Virtual Private Network)

### Concept

- Creates **encrypted tunnel** between client & server.
    
- Provides secure remote access, anonymity, and protection from intrusions.
    
- Common solutions: **OpenVPN, L2TP/IPsec, PPTP, SSTP, SoftEther**.
    
- Widely used by organizations for secure employee access.
    

### OpenVPN

- **Install**:
    

bash

```
sudo apt install openvpn -y
```

- **Config file**: `/etc/openvpn/server.conf`
    
- Features: encryption, tunneling, traffic shaping, routing.
    

### Connect to VPN

bash

```
sudo openvpn --config internal.ovpn
```

- After connection → communicate with internal hosts securely.
    

## 🧠 Key Takeaways

- **NFS**: convenient file sharing, but misconfigurations = security risk.
    
- **Web Servers**: Apache for full hosting, Python server for quick transfers.
    
- **VPNs**: secure remote access, essential for admins & penetration testers.
