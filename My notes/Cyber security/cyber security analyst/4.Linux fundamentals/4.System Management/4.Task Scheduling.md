## 📘 Concept

- Automates tasks at specific times/intervals.
    
- Common uses: updates, backups, script execution, DB maintenance.
    
- Ensures consistency, reliability, and can send alerts.
    
- Important for **cybersecurity**: attackers may hide malicious cron jobs or timers.
    

**Analogy:** Like setting a coffee machine to brew automatically every morning.

## ⚙️ Systemd Timers

### Steps:

1. **Create Timer** → defines when task runs.
    
2. **Create Service** → defines what task executes.
    
3. **Reload systemd** → apply changes.
    
4. **Start & Enable Timer** → activate automation.
    

### Example: `mytimer.timer`

ini

```
[Unit]
Description=My Timer

[Timer]
OnBootSec=3min        # Run once after boot
OnUnitActiveSec=1h    # Run every hour

[Install]
WantedBy=timers.target
```

### Example: `mytimer.service`

ini

```
[Unit]
Description=My Service

[Service]
ExecStart=/full/path/to/my/script.sh

[Install]
WantedBy=multi-user.target
```

### Commands:

bash

```
sudo systemctl daemon-reload
sudo systemctl start mytimer.timer
sudo systemctl enable mytimer.timer
```

## ⏰ Cron Jobs

### Structure (crontab fields):

Code

```
Minute (0-59) | Hour (0-23) | Day (1-31) | Month (1-12) | Weekday (0-7) | Command
```

### Examples:

cron

```
# System Update every 6 hours
0 */6 * * * /path/to/update_software.sh

# Run scripts on 1st day of month at midnight
0 0 1 * * /path/to/scripts/run_scripts.sh

# Cleanup DB every Sunday at midnight
0 0 * * 0 /path/to/scripts/clean_database.sh

# Backup every Sunday at midnight
0 0 * * 7 /path/to/scripts/backup.sh
```

- Logs & notifications can be configured for monitoring.
    

## 🔍 Systemd vs Cron

|Feature|Systemd Timers|Cron Jobs|
|---|---|---|
|Setup|Create timer + service files|Edit crontab file|
|Flexibility|Event-based triggers (boot, active)|Time-based only|
|Use Cases|Complex automation, dependencies|Simple repetitive scheduling|
|Example|`systemctl enable mytimer.timer`|`crontab -e`|

**Systemd Flow:** Timer → triggers Service → executes Script → logs via systemd



#### Create a Timer

To create a timer for systemd, we need to create a directory where the timer script will be stored.

`Abhishek27@htb[/htb]$ sudo mkdir /etc/systemd/system/mytimer.timer.d Abhishek27@htb[/htb]$ sudo vim /etc/systemd/system/mytimer.timer`

```
[Unit]
Description=My Timer

[Timer]
OnBootSec=3min
OnUnitActiveSec=1hour

[Install]
WantedBy=timers.target
```


#### Create a Service

`Abhishek27@htb[/htb]$ sudo vim /etc/systemd/system/mytimer.service`

```
[Unit]
Description=My Service

[Service]
ExecStart=/full/path/to/my/script.sh

[Install]
WantedBy=multi-user.target
```
#### Reload Systemd

        shellsession
`Abhishek27@htb[/htb]$ sudo systemctl daemon-reload`

#### Start the Timer & Service

        shellsession
