## 🌟 Why It Matters

User management ensures that each person on a system has the **right access** to files, directories, and commands. It’s essential for:

- **Security** → Prevent unauthorized access.
    
- **Collaboration** → Assign users to groups for shared resources.
    
- **Administration** → Allow elevated tasks without logging in as root.
    

## 📌 Key Commands

|Command|Description|
|---|---|
|`sudo`|Execute a command as another user (commonly root).|
|`su`|Switch user (default: superuser).|
|`useradd`|Create a new user.|
|`userdel`|Delete a user account.|
|`usermod`|Modify an existing user.|
|`addgroup`|Create a new group.|
|`delgroup`|Remove a group.|
|`passwd`|Change a user’s password.|

## ⚙️ Examples

### 1. **Execution as a user**

bash

```
cat /etc/shadow
```

→ Permission denied (only root can read).

### 2. **Execution as root with sudo**

bash

```
sudo cat /etc/shadow
```

→ Displays encrypted password entries.

### 3. **Create a new user (Alex)**

bash

```
sudo useradd alex
```

### 4. **Add Alex to a group (developers)**

bash

```
sudo usermod -aG developers alex
```

### 5. **Delete a user**

bash

```
sudo userdel alex
```

### 6. **Change password**

bash

```
passwd alex
```

Code

```
   ├── Create User (useradd)

   ├── Delete User (userdel)
   ├── Manage Groups (addgroup/delgroup)
   └── Execute as User (sudo/su)
```

## 📝 Key Takeaways

- **sudo** is safer than logging in as root directly.
    
- Users belong to **groups**, which define shared permissions.
    
- `/etc/passwd` stores user info, `/etc/shadow` stores encrypted passwords.
    
- Practice commands in a safe environment to build confidence.

# 🧑‍💻 Practice Scenario: Onboarding Alex

## 1. **Create a new user account**

bash

```
sudo useradd -m alex
```

- `-m` ensures a home directory (`/home/alex`) is created.
    

## 2. **Set a password for Alex**

bash

```
sudo passwd alex
```

- You’ll be prompted to enter and confirm a password.
    

## 3. **Create a new group (developers)**

bash

```
sudo addgroup developers
```

## 4. **Add Alex to the group**

bash

```
sudo usermod -aG developers alex
```

- `-aG` adds Alex to the group without removing existing group memberships.
    

## 5. **Verify group membership**

bash

```
groups alex
```

- Output will show Alex’s groups (e.g., `alex : alex developers`).
    

## 6. **Create a project directory for the group**

bash

```
sudo mkdir /projects
sudo chown root:developers /projects
sudo chmod 770 /projects
```

- Only members of `developers` can read/write/execute inside `/projects`.
    

## 7. **Switch to Alex’s account**

bash

```
su - alex
```

- You’ll now be logged in as Alex.
    

## 8. **Test access**

bash

```
cd /projects
touch testfile.txt
```

- If permissions are correct, Alex can create files inside `/projects`.
    

## 9. **Execute a command with elevated privileges**

bash

```
sudo ls /root
```

- Alex will need to be in the `sudoers` file to run this.
    
- If not, you can add Alex by editing:
    
    bash
    
    ```
    sudo visudo
    ```
    
    and adding:
    
    Code
    
    ```
    alex ALL=(ALL:ALL) ALL
    ```

## 🎯 Key Takeaways

- **useradd + passwd** → create account.
    
- **usermod -aG** → assign groups.
    
- **chown/chmod** → control directory access.
    
- **sudo/su** → execute as another user.
