## 🔹 Step 1: Connect to the Target via SSH

If you’ve spawned a lab machine (like in HackTheBox or TryHackMe), connect with:

bash

```
ssh htb-student@<IP address>
```

- Replace `<IP address>` with the machine’s actual IP.
    
- If prompted, accept the fingerprint and enter the password provided by the lab.
    

## 🔹 Step 2: Basic Situational Awareness Commands

Run these one by one and observe the output:

- **Who am I?**
    
    bash
    
    ```

- **User identity & groups**
    
    bash
    
    ```

- **Hostname**
    
    bash
    
    ```

- **System info**
    
    bash
    
    ```

- **Kernel release only**
    
    bash
    
    ```
    uname -r
    ```
    
    → Useful for checking if the kernel has known exploits.

## 🔹 Step 5: Networking

- **Interfaces & IPs**
    
    bash
    
    ```
    ifconfig
    ```
    
    _(On modern systems, use_ `ip a` _instead.)_
    
- **Routing & sockets**
    
    bash
    
    ```
    ip route
    ss -tuln
    netstat -tuln
    ```
    
**`ip route`**

Shows the **routing table** — how your system decides where to send network traffic.

```
default via 192.168.1.1 dev eth0    ← Default gateway (internet traffic goes here)
192.168.1.0/24 dev eth0             ← Local network reachable via eth0
```

Think of it like a **GPS map** — it tells packets _which road to take_ to reach a destination.

---

**`ss -tuln`**

Shows **open/listening network sockets** (modern replacement for `netstat`).

|Flag|Meaning|
|---|---|
|`-t`|TCP sockets|
|`-u`|UDP sockets|
|`-l`|Only _listening_ ports|
|`-n`|Show numbers, not names|

```
tcp  LISTEN  0.0.0.0:22    ← SSH is listening on port 22
tcp  LISTEN  127.0.0.1:80  ← Web server listening locally
```

Think of it as **"which doors are open"** on your machine — useful to see what services are running and on which ports.

---

**`netstat -tuln`**

Does the **same thing as `ss -tuln`** — shows TCP/UDP listening ports with numeric output.

It's the **older tool** (from `net-tools` package). `ss` is faster and preferred on modern systems, but `netstat` is still widely used and you'll see it in older tutorials/docs.

---

**Quick Summary Table**

|Command|Purpose|Think of it as|
|---|---|---|
|`ip route`|Routing table — where traffic is sent|Road map for packets|
|`ss -tuln`|Open ports & listening services (modern)|Which doors are open|
|`netstat -tuln`|Same as above (older tool)|Same, older version|

---

These three together are go-to commands for **network troubleshooting** — great stuff for your HTB/cybersecurity work! 🔍

Here's a breakdown of key Linux commands for system reconnaissance:

---

### 🔍 System Information Commands

|Command|Description|
|---|---|
|`uname -m`|Prints the **machine hardware name** (e.g., `x86_64`)|
|`uname -r`|Prints the **kernel release** version (e.g., `4.15.0-123-generic`)|
|`uname -a`|Prints **all system info**: kernel name, hostname, release, version, hardware, OS|
|`uname -p`|Prints the **processor type**|

---

### 👤 User Information Commands

|Command|Description|
|---|---|
|`grep htb-student /etc/passwd`|Shows the full `/etc/passwd` entry for a user, including **UID, GID, home dir, and shell**|
|`getent passwd htb-student`|Fetches user info from the system database (works even with LDAP/NIS)|
|`echo $MAIL`|Displays the **mail path** environment variable for the current user|
|`cat /etc/passwd`|Lists **all users** on the system|

---

### 🌐 Network Interface Commands

|Command|Description|
|---|---|
|`ip link show`|Shows all **network interfaces** with their MTU, state, and MAC address|
|`ip addr show`|Shows interfaces with **IP addresses** and MTU values|
|`ifconfig`|Classic command to display **network interface configuration** (IP, MTU, RX/TX stats)|
|`ip route`|Shows the **routing table**|

---

### 📬 Mail Path

|Path|Description|
|---|---|
|`/var/mail/<username>`|Default **mail spool** location on most Linux systems|
|`/var/spool/mail/<username>`|Alternative mail path (symlinked to `/var/mail` on many distros)|
|`$MAIL`|Environment variable pointing to the user's **mail file**|

---

### 🔑 Key `/etc/passwd` Field Format

```
username:password:UID:GID:comment:home_dir:shell
htb-student:x:1000:1000::/home/htb-student:/bin/bash
```

| Field      | Meaning                          |
| ---------- | -------------------------------- |
| `username` | Login name                       |
| `x`        | Password stored in `/etc/shadow` |
| `UID`      | User ID number                   |
| `GID`      | Group ID number                  |
| `comment`  | Optional description             |
| `home_dir` | User's home directory            |
| `shell`    | Default shell on login           |
