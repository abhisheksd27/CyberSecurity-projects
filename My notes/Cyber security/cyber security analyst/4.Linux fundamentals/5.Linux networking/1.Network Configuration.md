## 📘 Why It Matters

- Essential for penetration testers → set up test environments, manipulate traffic, exploit vulnerabilities.
    
- Involves: managing interfaces, assigning IPs, configuring gateways/DNS, monitoring, troubleshooting, and hardening.
    
- Protocols: **TCP/IP, DNS, DHCP, FTP**.
    

## 🔧 Configuring Network Interfaces

### View Interfaces

- **ifconfig** (older, still used)
    
- **ip addr** (modern replacement, more features)
    

bash

```
ifconfig
ip addr
```

### Activate Interface

bash

```
sudo ifconfig eth0 up
sudo ip link set eth0 up
```

### Assign IP Address

bash

```
sudo ifconfig eth0 192.168.1.2
```

### Assign Netmask

bash

```
sudo ifconfig eth0 netmask 255.255.255.0
```

### Set Default Gateway

bash

```
sudo route add default gw 192.168.1.1 eth0
```

### Configure DNS

Edit `/etc/resolv.conf`:

txt

```
nameserver 8.8.8.8
nameserver 8.8.4.4
```

⚠️ Not persistent → use `/etc/network/interfaces` for permanent settings.

### Persistent Settings Example

txt

```
auto eth0
iface eth0 inet static
  address 192.168.1.2
  netmask 255.255.255.0
  gateway 192.168.1.1
  dns-nameservers 8.8.8.8 8.8.4.4
```

Restart networking:

bash

```
sudo systemctl restart networking
```

## 🔒 Network Access Control (NAC)

### Models

- **DAC (Discretionary Access Control)** → owner sets permissions.
    
- **MAC (Mandatory Access Control)** → OS enforces strict rules (used in military/gov).
    
- **RBAC (Role-Based Access Control)** → permissions based on roles, scalable for large orgs.
    

### Tools

- **SELinux** → kernel-level MAC, fine-grained but complex.
    
- **AppArmor** → profile-based MAC, simpler to use.
    
- **TCP Wrappers** → restrict service access by IP.
    

## 📊 Monitoring Tools

- **syslog/rsyslog** → system logs.
    
- **ss** → socket statistics.
    
- **lsof** → list open files.
    
- **ELK stack** → advanced traffic analysis.
    
- **Wireshark/tshark/tcpdump** → packet capture.
    

## 🛠️ Troubleshooting Tools

- **ping** → test connectivity.
    

bash

```
ping 8.8.8.8
```

- **traceroute** → trace packet path.
    

bash

```
traceroute www.inlanefreight.com
```

- **netstat** → view active connections.
    

bash

```
netstat -a
```

- **nmap** → scan ports/services.
    
- **tcpdump/Wireshark** → capture/analyze packets.
    

### Common Issues

- Connectivity problems.
    
- DNS resolution failures.
    
- Packet loss.
    
- Misconfigured firewalls/routers.
    
- Hardware failures or congestion.
    

## 🛡️ Hardening Linux Networks

### SELinux

- Kernel-integrated MAC.
    
- Fine-grained policies for processes/files.
    
- Strong but complex to configure.
    

### AppArmor

- Profile-based MAC.
    
- Easier to configure, user-friendly.
    

### TCP Wrappers

- Restrict access to services by IP.
    
- Simple, effective for basic protection.
    

## 🧠 Key Takeaways

- **ifconfig/ip** → configure interfaces.
    
- **route + resolv.conf** → set gateway & DNS.
    
- **/etc/network/interfaces** → persistent settings.
    
- NAC models (DAC, MAC, RBAC) secure access.
    
- Monitoring & troubleshooting tools detect issues.
    
- Hardening with SELinux, AppArmor, TCP Wrappers reduces risk.
