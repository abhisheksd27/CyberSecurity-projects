---

### 📁 Linux Navigation Commands

|Command|Description|
|---|---|
|`pwd`|**Print Working Directory** — shows the full path of the directory you are currently in|
|`ls`|**List** — displays all files and directories in the current directory|
|`ls -l`|List with **detailed info**: permissions, links, owner, group, size, date, and name|
|`ls -la`|List **all files including hidden ones** (files starting with `.`) with detailed info|
|`ls -l /path/`|List contents of a **specific directory** without navigating to it|
|`cd /path/`|**Change Directory** — navigate to a specified directory|
|`cd -`|Jump back to the **previous directory** you were in|
|`cd ..`|Move **up one level** to the parent directory|
|`cd shm && clear`|**Chain commands** — change directory and then clear the terminal|
|`clear`|**Clear** the terminal screen of all previous output|

---

### 🔑 Special Directory Symbols

|Symbol|Meaning|
|---|---|
|`.`|Refers to the **current directory**|
|`..`|Refers to the **parent directory**|

---

### ⌨️ Useful Shell Shortcuts

|Shortcut|Description|
|---|---|
|`[TAB]` (once)|**Auto-completes** a command or path|
|`[TAB][TAB]` (twice)|Shows **all possible matches** for the current input|
|`[Ctrl] + [L]`|**Clears the terminal** (same as `clear` command)|
|`[Ctrl] + [R]`|**Search command history** — type to find previously used commands|
|`↑ / ↓` arrow keys|**Scroll through command history**|

---

### 📋 `ls -l` Output Column Breakdown

|Column|Description|
|---|---|
|`drwxr-xr-x`|**Type and permissions** (d = directory, - = file)|
|`2`|Number of **hard links**|
|`cry0l1t3`|**Owner** of the file/directory|
|`htbacademy`|**Group owner**|
|`4096`|**Size** in bytes (or blocks used for directories)|
|`Nov 13 17:37`|**Last modified** date and time|
|`Desktop`|**File/Directory name**|

### 💡 Key Concepts

- **Hidden files** start with a dot (e.g., `.bashrc`, `.bash_history`) and are only visible with `ls -la`
- **Auto-complete** with `[TAB]` speeds up navigation significantly
- You don't need to `cd` into a directory to list its contents — just pass the path to `ls`
- Chaining commands with `&&` runs the second command only if the first succeeds
  
  
  On Linux, every file has an **inode number** (sometimes called an index number). This is how the filesystem internally keeps track of files. The “sudoers” file in `/etc` is no exception.

Here’s how you can check its inode/index number:

## 🔍 Step-by-Step

1. **Navigate to** `/etc` **directory**
    
    bash
    
    ```
    cd /etc
    ```
    
2. **List files with their inode numbers**
    
    bash
    
    ```
    ls -i sudoers
    ```
    
    - The `-i` option shows the inode number before the filename.
        
    - Example output:
        
        Code
        
        ```
        123456 sudoers
        ```
        
        Here, `123456` is the inode number.
        
3. **Alternative: Use** `stat` **command**
    
    bash
    
    ```
    stat /etc/sudoers
    ```
    
    - This gives detailed info about the file, including its inode number.
        
    - Look for the line:
        
        Code
        
        ```
        Inode: 123456
        ```
        

## 🛡️ Cybersecurity Angle

- **Why it matters:**
    
    - Inodes are useful when investigating file tampering. Even if a file is renamed, its inode stays the same.
        
    - Attackers sometimes replace or symlink files; defenders can check inode numbers to confirm authenticity.
        
    - Forensic analysts use inode numbers to track deleted or hidden files.
