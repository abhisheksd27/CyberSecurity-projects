# 🔷 1. What is WMI? (Simple Understanding)

👉 **WMI = Windows internal management system**

It allows you to:

- Monitor systems
- Control systems
- Automate tasks
- Execute commands (locally & remotely)

📌 Think of WMI as:

"API + Database + Control Layer for Windows"

---

# 🧠 2. Why WMI is Important

### ✔ Used by:

- System Administrators (automation)
- Security Engineers (monitoring)
- Attackers (exploitation & lateral movement)

### 🔥 Key Capabilities:

- Get system info
- Run commands remotely
- Modify configurations
- Schedule tasks

---

# 🧩 3. WMI Architecture (Core Components)

Here’s the simplified flow:

User / Script  
     ↓  
WMI Consumer (request)  
     ↓  
CIM Object Manager  
     ↓  
WMI Service (Winmgmt)  
     ↓  
WMI Providers  
     ↓  
System Hardware / Software

---

## 🔹 Components Explained

### 1. WMI Service

- Runs in background
- Acts like a **middleman**

---

### 2. Managed Objects

- Anything you can control:
    - CPU
    - Disk
    - Processes
    - Network

---

### 3. WMI Providers

- Collect real data from system

---

### 4. Classes

- Blueprint of data

📌 Example:

Win32_OperatingSystem  
Win32_Process  
Win32_ComputerSystem

---

### 5. Methods

- Actions you can perform

📌 Example:

- Start process
- Stop service
- Rename file

---

### 6. WMI Repository

- Stores system data (like database)

---

### 7. CIM Object Manager

- Handles requests and responses

---

### 8. WMI API

- Interface for apps/scripts

---

### 9. WMI Consumer

- The one making request (you / script)

---

# 💻 4. Using WMI in Command Line

## 🔹 WMIC (Old but Important)

⚠️ Deprecated but still asked in interviews

### Example:

wmic computersystem get name

### OS Info:

wmic os list brief

---

# ⚡ 5. Using WMI in PowerShell (Modern Way)

## 🔹 Get System Info

Get-WmiObject -Class Win32_OperatingSystem

### Filter Output:

Get-WmiObject -Class Win32_OperatingSystem |   
select SystemDirectory, Version

---

## 🔹 Common Classes You MUST Know

|Class|Purpose|
|---|---|
|Win32_OperatingSystem|OS info|
|Win32_Process|Running processes|
|Win32_Service|Services|
|Win32_ComputerSystem|System details|

---

# ⚙️ 6. Performing Actions (Methods)

### 🔹 Example: Rename File

Invoke-WmiMethod -Path "CIM_DataFile.Name='C:\file.txt'" `  
-Name Rename -ArgumentList "C:\newfile.txt"

👉 ReturnValue = 0 → Success ✅

---

# 🌐 7. Remote Execution (VERY IMPORTANT 🔥)

WMI can execute commands on remote systems:

Get-WmiObject -Class Win32_Process -ComputerName TARGET
