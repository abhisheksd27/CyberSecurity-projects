- Services run programs in the background
- **Service permissions = who can:**
    - Start/stop service
    - Modify service
    - Change executable path

👉 If misconfigured → attacker can:

- Run malicious code
- Escalate privileges
- Maintain persistence

---

# ⚠️ Why Service Permissions are Dangerous

## 🔥 Common Issues:

- Weak permissions set by:
    - Admin mistakes ❌
    - Third-party software ❌

---

## 📌 Real Example (Important)

- Service runs as user **Bob**
- Bob leaves company → account disabled

👉 Result:

- Service fails ❌
- Network issues (e.g., DHCP stops)

👉 Solution:

- Use **Service Accounts** (dedicated accounts)

---

# 👤 Service Accounts

## 📌 What are they?

- Special accounts used only for services

## 🧠 Types:

|Account|Description|
|---|---|
|LocalSystem|Highest privileges 🔥|
|LocalService|Limited access|
|NetworkService|Network access|

---

## ⚠️ Important

- Many services run as **LocalSystem**  
    👉 If hacked → full system control 💀

---

# ⚙️ Service Properties (Important Fields)

## 📌 Key Things to Check:

### 1️⃣ Service Name

- Used in command line

---

### 2️⃣ Path to Executable

C:\Windows\System32\svchost.exe

👉 If folder is writable:

- Replace file → run malware ❌

---

### 3️⃣ Log On As

- Which account runs service

👉 Example:

- LocalSystem → high privilege

---

### 4️⃣ Recovery Options

- What happens if service fails

👉 Example:

- Run a program after crash  
    👉 Attacker can:
- Execute malicious program

---

# 💻 Managing Services (sc command)

## 📌 View Service Config

sc qc wuauserv

👉 Shows:

- Path
- Startup type
- Account

---

## 📌 Start / Stop Service

sc stop wuauserv

👉 Requires admin ⚠️

---

## 📌 Change Executable Path (VERY IMPORTANT)

sc config wuauserv binPath=C:\malicious.exe

👉 If allowed:

- Malware runs as service 🔥

---

# 🔍 Checking Service Permissions

## 📌 Using sc sdshow

sc sdshow wuauserv

👉 Output:

- Looks complex 😵
- Uses **SDDL (Security Descriptor Definition Language)**

---

## 🧠 Important Concepts

### DACL (Discretionary Access Control List)

- Controls **who can access**

### SACL

- Logs access attempts

---

## 📌 Example Meaning

(A;;RPWP;;;AU)

- A → Allow
- AU → Authenticated Users
- RP → Start service
- WP → Write permissions

👉 Meaning:

- Any user can modify service ⚠️

---

# 💻 Using PowerShell

## 📌 Check Permissions

Get-Acl -Path HKLM:\System\CurrentControlSet\Services\wuauserv

👉 Shows:

- Users
- Permissions
- SDDL

---

# 🔥 Pentester Attack Scenarios

## 🎯 1. Weak Service Permissions

- User can modify service  
    👉 Change path → run malware

---

## 🎯 2. Writable Service Directory

- Replace executable file

👉 Example:

C:\Program Files\App\service.exe

If writable → replace file

---

## 🎯 3. Abuse Recovery Option

- Set service to run malicious script

---

## 🎯 4. Run as LocalSystem

- Malware runs as SYSTEM → full access

---

# ⚠️ Key Security Concept

## 🧠 Principle of Least Privilege

- Give **minimum permissions needed**

👉 Example:

- Service should NOT run as admin if not required

---

# 🧾 Final Summary (Quick Revision)

- Service permissions control service behavior
- Misconfigurations = 🔥 privilege escalation
- Important checks:
    - Executable path
    - Writable folders
    - Service account
- Tools:
    - `sc`
    - PowerShell
