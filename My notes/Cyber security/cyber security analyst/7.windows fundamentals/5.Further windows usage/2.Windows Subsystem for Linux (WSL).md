# 🔷 1. What is WSL?

👉 **Windows Subsystem for Linux (WSL)** lets you run **Linux directly inside Windows** — without dual boot or a full VM.

📌 Think of it as:

"Linux inside Windows (lightweight + fast)"

---

# ⚙️ 2. Why WSL Was Created

### 🔹 Problem:

- Developers needed Linux tools on Windows

### 🔹 Solution:

- Run Linux tools natively

---

# 🧠 3. WSL Versions (VERY IMPORTANT 🔥)

## 🔹 WSL 1

- No real Linux kernel
- Translates Linux → Windows system calls

## 🔹 WSL 2 (Modern & Recommended)

- Uses a **real Linux kernel**
- Runs via lightweight virtualization

👉 Uses features from Microsoft Hyper-V

---

# 💻 4. How to Install WSL

### 🔹 Command (Run as Admin):

Enable-WindowsOptionalFeature -Online -FeatureName Microsoft-Windows-Subsystem-Linux

### 🔹 Then:

- Install distro from Microsoft Store (Ubuntu, Kali, etc.)

---

# 🐧 5. Running Linux in Windows

### 🔹 Start Linux shell:

bash

👉 You get a full Linux terminal

---

# 📁 6. Linux File System Inside WSL

### 🔹 Standard Linux structure:

/  
├── bin  
├── etc  
├── home  
├── usr

---

# 🔄 7. Access Windows Files from WSL

### 🔹 Windows drives are mounted here:

/mnt/c  
/mnt/d

👉 Example:

cd /mnt/c/Users

---

# 🔁 8. Seamless Integration (Very Powerful)

You can:

- Run Linux commands in Windows
- Access Windows files from Linux
- Run Windows commands from Linux

---

# 🧪 9. Example Commands

### 🔹 Check system:

uname -a

### 🔹 List files:

ls /

### 🔹 Install packages:

sudo apt update

---

# 🚀 10. Why WSL is Powerful

## ✔ For Developers

- Run Python, Node, Docker
- Use Linux tools (grep, awk, sed)

## ✔ For Cybersecurity 🔥

- Use Kali tools
- Run scanning tools
- Practice Linux commands

---

# 🔐 11. Security Perspective (IMPORTANT 🔥)

### ✔ Benefits:

- Safe testing environment
- No need for full VM

### ❗ Risks:

- Shared file system access
- Can be abused for:
    - Running hidden scripts
    - Bypassing restrictions

---

# ⚖️ 12. WSL vs Virtual Machine

|Feature|WSL|Virtual Machine|
|---|---|---|
|Speed|Fast ⚡|Slower|
|Resource Usage|Low|High|
|Full OS|Partial (WSL1) / Full kernel (WSL2)|Full OS|
|Setup|Easy|Complex|
