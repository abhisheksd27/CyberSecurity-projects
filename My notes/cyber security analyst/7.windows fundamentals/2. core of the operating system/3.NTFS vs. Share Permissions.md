- Windows is used by **70%+ systems**
- Most attacks target:
    - File sharing
    - Misconfigured permissions

👉 Example:

- Malware spreading via **network shares (SMB)**
- Vulnerability like **EternalBlue**

# ⚔️ NTFS vs Share Permissions (Core Concept)

## 📌 Key Idea:

👉 Both permissions apply **together**

|Type|Applies When|
|---|---|
|NTFS|Local + Remote access|
|Share|Only over network (SMB)|

---

# 📁 Share Permissions

## 📌 Used when:

- Accessing folder via **network**

## 📊 Types:

|Permission|Meaning|
|---|---|
|Full Control|Everything + change permissions|
|Change|Read, write, delete|
|Read|Only view|

---

## 📌 Example

- You access:

\\192.168.1.10\CompanyData

👉 Share permission decides:

- Can you access or not

---

# 📁 NTFS Permissions

## 📌 Used when:

- Accessing files:
    - Locally 💻
    - Remotely 🌐

## 📊 Types:

|Permission|Meaning|
|---|---|
|Full Control|Everything|
|Modify|Edit + delete|
|Read & Execute|Open + run|
|Read|View|
|Write|Edit|

---

## 📌 Advanced (Special Permissions)

- Traverse folder
- Create files
- Delete
- Take ownership

👉 More **granular control** than share permissions

---

# ⚠️ IMPORTANT RULE (VERY IMPORTANT)

## 🧠 Final Access = MOST RESTRICTIVE PERMISSION

👉 Example:

|Permission Type|Access|
|---|---|
|Share|Full Control|
|NTFS|Read|

✅ Final Result = **Read only**

---

## 📌 Simple Example

- Share → Full access
- NTFS → No write

👉 You **cannot write** ❌

---

# 🏢 Real-World Scenario

## 📌 Situation:

- Folder shared with:
    - Everyone → Read (Share)
- NTFS:
    - User → Full Control

👉 Result:

- User can only **READ** (because Share restricts)

---

# 🔥 Pentester Perspective

## 🔍 What to Check

- Share permissions:
    - Everyone = Full Control ❌
- NTFS permissions:
    - Weak restrictions

---

## ⚠️ Common Misconfiguration

- Share = Full Control
- NTFS = Full Control

👉 Result:

- Anyone can:
    - Upload malware
    - Delete files
    - Exfiltrate data

---

# 💻 Important Commands

## 📌 List Shares (Linux)

smbclient -L IP -U username

---

## 📌 Connect to Share

smbclient \\IP\Share -U username

---

## 📌 Mount Share (Linux)

mount -t cifs //IP/Share /mnt -o username=user,password=pass

---

## 📌 View Shares (Windows)

net share

👉 Example Output:

C$       C:\  
ADMIN$   C:\Windows

---

# ⚠️ Hidden Default Shares

## 📌 Automatically created:

- `C$` → Entire C drive
- `ADMIN$` → Windows folder

👉 Dangerous if misused ⚠️

---

# 🔥 Firewall Role

- Windows Firewall can:
    - Block SMB access
- Profiles:
    - Public
    - Private
    - Domain

👉 Best practice:

- Allow specific rules
- Do NOT disable firewall ❌

# 🧠 Workgroup vs Domain

|Type|Authentication|
|---|---|
|Workgroup|Local SAM database|
|Domain|Active Directory|

👉 Important for login & attacks

---

# 🧾 Final Summary (Quick Revision)

- SMB = file sharing protocol
- Share permissions:
    - Apply only over network
- NTFS permissions:
    - Apply always
- Final access:
    - **Most restrictive wins**
- Misconfigurations = big security risk
